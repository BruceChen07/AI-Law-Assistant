# 项目阶段路线图总览

## 目标
建立可持续迭代的合同审计系统研发路线图，确保阶段目标、交付物、验收标准一致。

## 阶段划分
- 阶段 A：MVP（法规导入 + 合同解析 + 基础审计 + 批注输出）
- 阶段 B：增强（风险分级 + 报告 + 批量审计 + 归档）
- 阶段 C：协作与治理（权限 + 整改闭环 + 追踪）

## 交付节奏
- 每个阶段完成后必须输出：设计方案、实现思路、验收清单、风险与应对
- 每个阶段结束后记录关键决策到 decisions/

## 统一验收原则
- 可用：核心流程可稳定运行
- 可追溯：审计结果可关联法规条款
- 可落地：产出文件满足业务使用习惯

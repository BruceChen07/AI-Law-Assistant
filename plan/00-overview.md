# 项目阶段路线图总览

## 目标
建立可持续迭代的合同审计系统研发路线图，覆盖合同审计、OCR增强解析与可配置LLM调用，确保阶段目标、交付物、验收标准一致。

## 阶段划分
- 阶段 A：MVP（法规导入 + 合同解析 + 基础审计 + 批注输出）
- 阶段 B：增强（风险分级 + 报告 + 批量审计 + 归档）
- 阶段 C：协作与治理（权限 + 整改闭环 + 追踪）

## 模块边界
- 合同审计：合同上传、解析、审计结果呈现
- 法规管理：Admin 侧法规导入、条款切分、元数据维护
- OCR 引擎：扫描 PDF 回退 OCR、多引擎选择
- 模型调用：LLM 配置中心与模型调用封装
- 审计引擎：规则 + 相似度匹配 + 证据关联
- 输出模块：风险清单与审计摘要
- 用户与权限：登录鉴权、角色权限、审计追踪

## 关键依赖
- 文档解析：python-docx / pypdf
- OCR：pytesseract / pdf2image + tesseract / poppler
- LLM：可配置的 OpenAI 兼容接口
- 向量模型：BGE-small
- 重排模型：Cross-Encode

## 交付节奏
- 每个阶段完成后必须输出：设计方案、实现思路、验收清单、风险与应对
- 每个阶段结束后记录关键决策到 decisions/

## 统一验收原则
- 可用：核心流程可稳定运行
- 可追溯：审计结果可关联法规条款
- 可落地：产出文件满足业务使用习惯

# 技术架构与模块边界

## 总体架构
- 前端：Web UI（用户交互）
- 后端：FastAPI API 服务
- AI能力：合同解析、法规匹配、风险识别、批注生成
- 数据层：法规库、合同库、审计记录库、文件存储

## 模块边界
- 法规管理：法规导入、条款切分、元数据维护、版本管理
- 合同解析：Word/PDF 解析、字段抽取、摘要校验
- 审计引擎：规则 + 相似度匹配 + 证据关联
- 输出模块：批注版 Word、审计报告、风险清单
- 用户与权限：登录鉴权、角色权限、审计追踪

## 关键依赖
- 文档解析：python-docx / pypdf
- 向量模型：BGE-small
- 重排模型：Cross-Encode
